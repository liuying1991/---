# 贾维斯记忆大脑系统设计

## 一、现有技术深度分析

### 1.1 MemGPT/Letta 架构分析

**核心创新**：虚拟上下文管理（OS启发）

```
┌─────────────────────────────────────────┐
│           Main Context (RAM)            │
│  ┌─────────┐ ┌─────────┐ ┌───────────┐  │
│  │ System  │ │ Working │ │  Current  │  │
│  │ Prompt  │ │ Memory  │ │  Context  │  │
│  └─────────┘ └─────────┘ └───────────┘  │
└─────────────────────────────────────────┘
                    ↕ 分页调度
┌─────────────────────────────────────────┐
│         External Context (Disk)         │
│  ┌─────────┐ ┌─────────┐ ┌───────────┐  │
│  │ Recall  │ │Archive  │ │  Files    │  │
│  │ Storage │ │ Memory  │ │           │  │
│  └─────────┘ └─────────┘ └───────────┘  │
└─────────────────────────────────────────┘
```

**优点**：
- 无限上下文能力
- LLM自主管理记忆
- 白盒设计，可解释

**局限**：
- 缺乏时间维度
- 关系建模不足
- 无遗忘机制

### 1.2 Zep 知识图谱方法

**核心创新**：时序感知动态知识图谱

```
G = (N, E, φ)

三层子图：
├── Episode Subgraph (事件层)
│   └── 对话片段、时间戳
├── Semantic Entity Subgraph (实体层)
│   └── 人物、地点、事物
└── Community Subgraph (社区层)
    └── 实体聚类、主题
```

**优点**：
- 结构化记忆
- 时序关系
- 实体关联

**局限**：
- 图构建开销大
- 需要额外实体抽取

### 1.3 Cognee 混合方法

**核心创新**：知识图谱 + 向量检索

```
记忆存储 → 实体抽取 → 图构建
    ↓
向量嵌入 → 语义检索
    ↓
混合查询 → 上下文整合
```

### 3.2 核心创新点

#### 创新1: 动态记忆压缩 (Dynamic Memory Compression)

```go
// 记忆压缩器
type MemoryCompressor interface {
    // 压缩对话历史为摘要
    Compress(messages []Message) (*MemorySummary, error)

    // 提取关键信息
    Extract(messages []Message) ([]KeyInfo, error)

    // 识别重要度
    Importance(message Message) float64
}
```

**原理**：使用LLM自动识别对话中的：
- 关键决策
- 用户偏好
- 重要事实
- 任务结果

#### 创新2: 多维关联网络 (Multi-dimensional Association Network)

```
传统知识图谱: 实体 --关系--> 实体

贾维斯关联网络:
├── 实体关联 (谁/什么)
├── 时间关联 (何时)
├── 因果关联 (为什么)
├── 情感关联 (感受)
└── 上下文关联 (场景)
```

#### 创新3: 自适应遗忘机制 (Adaptive Forgetting)

```
记忆强度 = f(重要性, 访问频率, 时间衰减, 情感权重)

S(t) = I × F × e^(-λt) × E

其中:
- I: 初始重要性 (0-1)
- F: 访问频率因子
- λ: 衰减系数
- E: 情感权重
- t: 时间

当 S(t) < θ (阈值) 时，触发遗忘
```

#### 创新4: 记忆索引系统 (Memory Index System)

```go
// 多维索引
type MemoryIndex struct {
    // 时间索引
    TimeIndex *BTreeIndex

    // 实体索引
    EntityIndex *HashIndex

    // 向量索引 (语义检索)
    VectorIndex *VectorDB

    // 重要性索引
    ImportanceIndex *PriorityQueue
}
```

## 五、实现策略

### 5.1 存储选择

| 层级 | 存储介质 | 理由 |
|------|----------|------|
| L0 | 内存 | 最快访问 |
| L1 | SQLite | 结构化查询 |
| L2 | SQLite + 文件 | 大量历史 |
| L3 | SQLite + 向量DB | 语义检索 |

### 5.2 核心接口

```go
// 记忆大脑接口
type MemoryBrain interface {
    // 记忆写入
    Remember(ctx context.Context, memory Memory) error

    // 记忆检索
    Recall(ctx context.Context, query Query) ([]Memory, error)

    // 记忆更新
    Update(ctx context.Context, id string, update Update) error

    // 记忆遗忘
    Forget(ctx context.Context, condition Condition) error

    // 记忆压缩
    Compress(ctx context.Context) error

    // 获取上下文
    GetContext(ctx context.Context, limit int) string
}
```
