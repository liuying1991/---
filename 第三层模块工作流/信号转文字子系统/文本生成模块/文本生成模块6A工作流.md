# 文本生成模块6A工作流

## 模块概述

### 模块名称
文本生成模块

### 所属子系统
信号转文字子系统

### 模块功能描述
文本生成模块负责根据认知决策子系统的指令，生成适合婴儿理解和互动的文本内容。该模块能够生成简单、清晰、富有情感色彩的文本，用于与婴儿进行互动，如讲故事、唱歌谣、安抚话语等。同时，该模块还能根据婴儿的年龄、情绪状态和认知水平，调整生成文本的复杂度和风格。

### 模块职责
1. 文本规划：根据互动目标和婴儿状态规划文本内容
2. 内容生成：生成适合婴儿的互动文本内容
3. 风格调整：根据婴儿特点调整文本风格和复杂度
4. 情感注入：为生成的文本注入适当的情感色彩
5. 多样性控制：控制生成文本的多样性和新颖性
6. 质量评估：评估生成文本的质量和适宜性

### 模块接口
- **输入接口**：互动指令、婴儿状态、生成参数
- **输出接口**：生成文本、文本标签、质量评分
- **依赖模块**：认知决策子系统
- **被依赖模块**：语音合成模块、交互表达子系统

## Ask阶段：需求分析

### 功能需求
1. 支持多种文本类型生成，包括故事、歌谣、安抚话语、互动问答等
2. 能够根据婴儿年龄（0-3岁）调整文本复杂度和词汇选择
3. 支持情感文本生成，能够生成带有不同情感色彩的文本
4. 提供个性化文本生成，能够根据婴儿偏好调整内容
5. 支持多轮对话文本生成，能够维持对话的连贯性
6. 提供文本多样性控制，避免重复和单调的内容
7. 支持实时文本生成，响应时间不超过300ms
8. 提供文本质量评估，确保生成内容的适宜性

### 非功能需求
1. **性能需求**：文本生成响应时间不超过300ms
2. **质量需求**：生成文本的适宜性评分达到85%以上
3. **多样性需求**：生成文本的多样性指标达到80%以上
4. **可扩展性需求**：支持新文本类型和风格的扩展
5. **资源需求**：内存占用不超过1GB，CPU占用不超过40%

### 约束条件
1. **技术约束**：基于Python开发，使用PyTorch或TensorFlow框架
2. **数据约束**：需要大量适合婴儿的文本数据进行模型训练
3. **内容约束**：生成内容必须符合儿童安全和教育标准
4. **时间约束**：开发周期不超过8周

### 用户场景
1. **场景1**：睡前故事生成 - 根据婴儿状态生成适合的睡前故事
2. **场景2**：安抚话语生成 - 生成安抚婴儿情绪的话语
3. **场景3**：互动问答生成 - 生成与婴儿互动的问答内容

### 验收标准
1. 文本生成响应时间不超过300ms
2. 生成文本的适宜性评分达到85%以上
3. 生成文本的多样性指标达到80%以上
4. 能够生成至少5种不同类型的文本
5. 能够支持至少3种不同年龄段的文本复杂度
6. 系统连续运行24小时无故障

1. **输入处理层**：负责处理输入指令和婴儿状态信息
2. **规划层**：负责规划文本内容、风格和结构
3. **生成层**：负责生成基础文本内容
4. **优化层**：负责优化文本质量、注入情感和调整风格
5. **输出层**：负责格式化输出和质量评估

### 数据模型
文本生成模块的数据模型包括：
1. **输入数据模型**：包含互动指令、婴儿状态、生成参数等
2. **规划数据模型**：包含文本主题、结构、风格等规划信息
3. **生成数据模型**：包含基础文本、候选文本、生成概率等
4. **优化数据模型**：包含优化后文本、情感标签、质量指标等
5. **输出数据模型**：包含最终文本、文本标签、质量评分等

2. **深度学习框架**：PyTorch - 灵活的深度学习框架
3. **预训练模型**：GPT-2/BART - 先进的文本生成模型
4. **NLP库**：NLTK/spaCy - 文本处理和分析
5. **部署方案**：ONNX Runtime - 高效模型推理

### 模块设计
1. **核心组件**：
   - 输入处理器：负责处理输入指令和状态信息
   - 文本规划器：负责规划文本内容和结构
   - 文本生成器：负责生成基础文本内容
   - 文本优化器：负责优化文本质量和风格
   - 情感注入器：负责为文本注入情感色彩
   - 质量评估器：负责评估生成文本的质量
   - 模型管理器：负责模型的加载和管理

2. **组件交互**：
   - 输入处理器将处理后的信息传递给文本规划器
   - 文本规划器将规划信息传递给文本生成器
   - 文本生成器将基础文本传递给文本优化器
   - 文本优化器将优化后的文本传递给情感注入器
   - 情感注入器将最终文本传递给质量评估器
   - 质量评估器输出最终结果

3. **数据流**：
   输入指令 → 输入处理 → 文本规划 → 文本生成 → 文本优化 → 情感注入 → 质量评估 → 输出文本

4. **控制流**：
   生成请求 → 模型管理器 → 输入处理器 → 文本规划器 → 文本生成器 → 文本优化器 → 情感注入器 → 质量评估器 → 返回结果

# 文本生成器实现
class TextGenerator:
    def __init__(self, base_model, story_model, rhyme_model, qa_model):
        self.base_model = base_model
        self.story_model = story_model
        self.rhyme_model = rhyme_model
        self.qa_model = qa_model
        self.tokenizer = AutoTokenizer.from_pretrained(base_model.config.name_or_path)
        self.optimizer = TextOptimizer()
        self.emotion_injector = EmotionInjector()
        self.quality_assessor = QualityAssessor()
        
    def generate(self, instruction, baby_state, generation_params):
        # 输入处理
        processed_input = self._process_input(instruction, baby_state, generation_params)
        
        # 文本规划
        text_plan = self._plan_text(processed_input)
        
        # 文本生成
        if text_plan["type"] == "story":
            generated_text = self._generate_story(text_plan)
        elif text_plan["type"] == "rhyme":
            generated_text = self._generate_rhyme(text_plan)
        elif text_plan["type"] == "qa":
            generated_text = self._generate_qa(text_plan)
        else:
            generated_text = self._generate_base(text_plan)
        
        # 文本优化
        optimized_text = self.optimizer.optimize(generated_text, text_plan)
        
        # 情感注入
        if "emotion" in text_plan and text_plan["emotion"]:
            emotion_text = self.emotion_injector.inject(optimized_text, text_plan["emotion"])
        else:
            emotion_text = optimized_text
        
        # 质量评估
        quality_score = self.quality_assessor.assess(emotion_text, text_plan)
        
        # 如果质量不达标，重新生成
        if quality_score < 0.7:
            return self.generate(instruction, baby_state, generation_params)
        
        return {
            "text": emotion_text,
            "type": text_plan["type"],
            "emotion": text_plan.get("emotion", None),
            "quality_score": quality_score,
            "timestamp": time.time()
        }

# 文本优化器实现
class TextOptimizer:
    def __init__(self):
        self.style_adjuster = StyleAdjuster()
        self.diversity_controller = DiversityController()
        
    def optimize(self, text, text_plan):
        # 风格调整
        if "style" in text_plan:
            text = self.style_adjuster.adjust(text, text_plan["style"])
        
        # 复杂度调整
        if "complexity" in text_plan:
            text = self._adjust_complexity(text, text_plan["complexity"])
        
        # 多样性控制
        if "diversity" in text_plan:
            text = self.diversity_controller.control(text, text_plan["diversity"])
        
        # 长度调整
        if "length" in text_plan:
            text = self._adjust_length(text, text_plan["length"])
        
        return text
    
    def _adjust_complexity(self, text, complexity_level):
        # 根据复杂度级别调整文本
        if complexity_level == "low":
            # 简化词汇和句子结构
            text = self._simplify_vocabulary(text)
            text = self._simplify_sentences(text)
        elif complexity_level == "medium":
            # 适度调整
            text = self._moderate_adjustment(text)
        elif complexity_level == "high":
            # 保持或增加复杂度
            text = self._maintain_complexity(text)
        
        return text
