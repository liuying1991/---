# AI编程大模型任务执行清单

## 任务接收阶段

### 1. 任务理解
- [ ] 仔细阅读用户提供的任务描述
- [ ] 识别任务的核心目标和要求
- [ ] 确定任务的优先级和截止时间
- [ ] 识别任务涉及的技术领域和知识

### 2. 上下文分析
- [ ] 查看第三层模块工作流总览.md，了解整体系统架构
- [ ] 确定任务所属的子系统和模块
- [ ] 理解模块与其他模块的依赖关系
- [ ] 查看相关模块的当前状态和进度

### 3. 任务分类
- [ ] 根据任务性质确定所属阶段（Ask/Analyze/Apply/Assess/Assist/Adjust）
- [ ] 确定任务类型（需求分析/架构设计/代码实现/测试评估/部署运维/优化迭代）
- [ ] 评估任务复杂度和工作量
- [ ] 识别任务的风险和挑战

## 常见问题处理

### 1. 需求不明确
- [ ] 查看相关需求文档和用户故事
- [ ] 分析上下文和依赖关系
- [ ] 提出澄清问题
- [ ] 基于最佳实践补充需求细节

### 2. 技术选型困难
- [ ] 查看技术栈规范和约束
- [ ] 分析各技术方案的优缺点
- [ ] 参考类似场景的实现
- [ ] 考虑长期维护和扩展性

### 3. 模块依赖冲突
- [ ] 查看模块接口文档
- [ ] 与相关模块负责人沟通
- [ ] 分析依赖关系和影响范围
- [ ] 协调解决冲突

### 4. 性能不达标
- [ ] 分析性能瓶颈
- [ ] 查看性能优化指南
- [ ] 实施针对性优化
- [ ] 重新测试和验证

### 5. 文档不完整
- [ ] 查看文档模板和规范
- [ ] 参考其他模块的文档
- [ ] 补充缺失的内容
- [ ] 确保文档的一致性
