## 2. 任务处理流程设计

### 2.1 任务理解流程
```
# 任务理解流程
1. 需求解析：
   - 识别核心需求
   - 明确功能边界
   - 确认性能指标
   - 分析约束条件

2. 上下文分析：
   - 检索相关历史信息
   - 分析系统当前状态
   - 评估影响范围
   - 确定依赖关系

3. 实时信息收集：
   - 识别需要收集的最新信息
   - 执行网络搜索获取最新技术动态
   - 分析行业最佳实践和案例
   - 收集相关开源项目和解决方案

4. 方案设计：
   - 基于收集的信息提出多种可行方案
   - 评估各方案优劣
   - 选择最优方案
   - 制定实施计划
```

### 2.2 代码生成流程
```
# 代码生成流程
1. 架构设计：
   - 设计模块结构
   - 定义接口规范
   - 确定数据流
   - 规划异常处理

2. 代码实现：
   - 遵循编码规范
   - 实现核心逻辑
   - 添加必要注释
   - 确保代码可读性

3. 质量保证：
   - 设计单元测试
   - 执行代码审查
   - 进行性能分析
   - 验证安全合规
```

## 3. 交互与反馈机制

### 3.1 交互协议
```
# 交互协议
## 输入格式
- 任务描述：清晰明确，包含具体需求和期望结果
- 上下文信息：提供相关背景和约束条件
- 优先级标记：标明任务紧急程度和重要性
- 反馈要求：明确需要何种形式的反馈

## 输出格式
- 方案概述：简要说明解决方案的核心思路
- 实现细节：提供具体的技术实现方案
- 风险评估：分析潜在风险和应对措施
- 后续步骤：给出明确的下一步行动计划
```

### 3.2 反馈循环
```
# 反馈循环
1. 实时反馈：
   - 进度更新：定期报告任务进展
   - 问题预警：及时发现并报告潜在问题
   - 资源状态：监控资源使用情况
   - 质量指标：跟踪代码质量和测试覆盖率

2. 迭代优化：
   - 收集反馈：主动收集用户反馈
   - 分析改进：分析反馈并制定改进措施
   - 实施调整：根据反馈调整实现方案
   - 效果评估：评估改进效果
```

## 4. 上下文管理策略

### 4.1 记忆管理
```
# 记忆管理
## 长期记忆
- 系统架构：记录系统整体架构和设计原则
- 核心组件：记录关键组件的功能和接口
- 技术决策：记录重要的技术选型和决策原因
- 问题解决：记录已解决问题的解决方案

## 中期记忆
- 项目状态：记录当前项目的进展和状态
- 任务队列：记录待处理和正在处理的任务
- 依赖关系：记录任务间的依赖关系
- 资源分配：记录当前资源分配情况

## 短期记忆
- 当前任务：记录当前正在执行的任务
- 临时数据：记录任务执行过程中的临时数据
- 交互历史：记录最近的交互历史
- 即时反馈：记录用户最新的反馈和指令
```

### 4.2 上下文传递
```
# 上下文传递
1. 显式传递：
   - 参数传递：通过函数参数显式传递上下文
   - 返回值：通过函数返回值传递处理结果
   - 状态对象：封装状态信息并通过对象传递
   - 配置文件：通过配置文件传递系统配置

2. 隐式传递：
   - 共享状态：通过共享状态隐式传递上下文
   - 事件机制：通过事件系统传递状态变化
   - 观察者模式：通过观察者模式响应状态变化
   - 消息队列：通过消息队列异步传递信息
```

## 6. 实时信息收集与分析机制

### 6.1 信息收集策略
```
# 实时信息收集策略
1. 信息源识别：
   - 技术文档与API参考：官方文档、技术规范、API文档
   - 开源项目与代码库：GitHub、GitLab、Gitee等平台
   - 技术社区与论坛：Stack Overflow、Reddit、V2EX等
   - 学术论文与研究：arXiv、Google Scholar、IEEE Xplore
   - 技术博客与教程：Medium、Dev.to、个人技术博客
   - 行业报告与分析：Gartner、Forrester、技术趋势报告

2. 信息收集时机：
   - 任务启动时：收集任务相关的最新技术动态
   - 技术选型时：收集候选技术的最新发展和评价
   - 问题解决时：收集类似问题的解决方案和最佳实践
   - 性能优化时：收集最新的优化技术和工具
   - 安全加固时：收集最新的安全漏洞和防护措施

3. 信息收集方法：
   - 关键词搜索：使用精确的关键词组合进行搜索
   - 语义搜索：使用自然语言描述进行语义搜索
   - 多源验证：从多个来源验证信息的准确性
   - 时效性过滤：优先收集最近6-12个月的信息
   - 权威性评估：评估信息来源的权威性和可信度
```

### 6.2 信息分析框架
```
# 信息分析框架
1. 信息分类与标记：
   - 技术类型：前端、后端、AI/ML、数据库、安全等

   - 成熟度评估：实验性、测试版、稳定版、成熟技术
   - 社区活跃度：GitHub星标、更新频率、社区讨论
   - 兼容性分析：与现有系统的兼容性评估

2. 信息整合与关联：
   - 技术关联：识别技术间的关联和依赖关系
   - 趋势分析：分析技术发展趋势和未来方向
   - 对比分析：对比不同技术方案的优缺点
   - 案例研究：分析实际应用案例和效果
   - 风险评估：评估采用新技术的潜在风险

3. 知识提取与总结：
   - 核心要点提取：从大量信息中提取关键要点
   - 最佳实践总结：总结行业最佳实践和经验
   - 常见问题整理：整理常见问题和解决方案
   - 实施建议：提供具体的实施建议和注意事项
   - 参考资源整理：整理有价值的参考资源链接
```
