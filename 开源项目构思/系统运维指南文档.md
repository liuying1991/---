## 2. 系统架构概述

### 2.1 系统组件架构
真实婴儿AI管家系统由以下核心组件构成：

1. **核心控制器层**
   - RealInfantAIButler：系统主控制器，协调各子系统工作
   - BrainCore：大脑核心，负责数据融合与决策制定

2. **感知系统层**
   - RealSensorySystem：感官系统，处理多模态感知数据
   - HumanPerceptionSystem：人类感知系统，识别情感与意图

3. **交互系统层**
   - InteractionSystemManager：交互系统管理器
   - EnhancedVocalOrgan：增强语音器官

4. **思维系统层**
   - ThinkingSystemManager：思维系统管理器
   - ReasoningEngine：推理引擎

5. **学习与进化层**
   - IntelligentEvolutionSystem：智能进化系统
   - InternetDataCollector：互联网数据收集器

6. **知识管理层**
   - KnowledgeGraph：知识图谱系统

### 2.2 系统部署架构
系统支持多种部署模式：
- 单机部署：适用于开发和小规模应用场景
- 分布式部署：适用于大规模生产环境
- 云原生部署：基于Kubernetes的容器化部署
- 混合云部署：结合本地和云资源的部署模式

### 2.3 数据流架构
系统数据流主要包括：
- 感知数据流：RealSensorySystem → HumanPerceptionSystem → BrainCore
- 交互数据流：BrainCore → InteractionSystemManager → EnhancedVocalOrgan
- 思维数据流：BrainCore → ThinkingSystemManager → ReasoningEngine
- 知识数据流：InternetDataCollector → KnowledgeGraph → BrainCore
- 进化数据流：BrainCore → IntelligentEvolutionSystem → BrainCore

### 3.3 系统安装步骤

```bash

# 3. 安装数据库

sudo systemctl enable redis-server
```

### 4.1 监控体系架构

1. **基础设施监控**：服务器硬件、网络、存储等

### 4.4 告警配置

#### 4.4.1 告警规则示例
```yaml
# prometheus-rules.yml示例
groups:
- name: infant-ai-alerts
  rules:
  - alert: HighCPUUsage
    expr: 100 - (avg by(instance) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 90
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "高CPU使用率告警"
      description: "实例 {{ $labels.instance }} CPU使用率超过90%，当前值: {{ $value }}%"

  - alert: HighMemoryUsage
    expr: (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100 > 90
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "高内存使用率告警"
      description: "实例 {{ $labels.instance }} 内存使用率超过90%，当前值: {{ $value }}%"

  - alert: APIHighErrorRate
    expr: rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m]) * 100 > 5
    for: 5m
    labels:
      severity: critical
    annotations:
      summary: "API高错误率告警"
      description: "API错误率超过5%，当前值: {{ $value }}%"

  - alert: APIHighResponseTime
    expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) > 1
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "API高响应时间告警"
      description: "API P95响应时间超过1秒，当前值: {{ $value }}秒"
```

### 5.2 数据备份策略

#### 5.2.1 数据库备份
```bash
#!/bin/bash
# 数据库备份脚本

# 配置参数
DB_HOST="localhost"
DB_PORT="5432"
DB_NAME="infant_ai_db"
DB_USER="infantai"
BACKUP_DIR="/backup/database"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/${DB_NAME}_backup_$DATE.sql"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 执行备份
pg_dump -h $DB_HOST -p $DB_PORT -U $DB_USER -d $DB_NAME > $BACKUP_FILE

# 压缩备份文件
gzip $BACKUP_FILE

# 删除7天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

echo "数据库备份完成: $BACKUP_FILE.gz"
```

#### 5.3.2 应用更新脚本
```bash
#!/bin/bash
# 应用更新脚本

# 配置参数
APP_DIR="/opt/infant-ai"
BACKUP_DIR="/backup/application"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/app_backup_$DATE.tar.gz"
SERVICE_NAME="infant-ai"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 停止服务
systemctl stop $SERVICE_NAME

# 备份当前应用
tar -czf $BACKUP_FILE -C $APP_DIR .

# 更新应用代码
cd $APP_DIR
git pull origin main

# 更新依赖
source venv/bin/activate
pip install -r requirements.txt

# 运行数据库迁移
python manage.py migrate

# 启动服务
systemctl start $SERVICE_NAME

# 检查服务状态
sleep 10
if systemctl is-active --quiet $SERVICE_NAME; then
    echo "应用更新成功，服务已启动"
else
    echo "应用更新失败，正在回滚..."
    # 回滚操作
    systemctl stop $SERVICE_NAME
    tar -xzf $BACKUP_FILE -C $APP_DIR
```

```python
    ADMIN = "admin"
    OPERATOR = "operator"
    VIEWER = "viewer"

def require_role(role):
    def decorator(f):
        @wraps(f)
        def decorated_function(*args, **kwargs):
            user = get_current_user()  # 获取当前用户
            if not user or user.role != role:
                return jsonify({"msg": "权限不足"}), 403
            return f(*args, **kwargs)
        return decorated_function
    return decorator

@app.route('/admin/dashboard')
@require_role(UserRole.ADMIN)
def admin_dashboard():
    return render_template('admin_dashboard.html')

@app.route('/operator/controls')
@require_role(UserRole.OPERATOR)
def operator_controls():
    return render_template('operator_controls.html')
```
